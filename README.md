# ARMA-GARCH Beta Risk Model

A professional-grade next-day equity risk model built on 
**ARMA(1,1)-GARCH(1,1)** filtering with **Normal Inverse Gaussian (NIG)** 
heavy-tailed innovation modelling. The model produces daily **Value at Risk 
(VaR)** and **Conditional Value at Risk (CVaR)** forecasts and is validated 
across five structurally distinct asset classes using a comprehensive 
four-test statistical assessment suite.

**Model verdict: PASS - deploy ready across all asset classes.**

## Methodology
```
Raw prices
    ↓
Log returns  r_t = log(S_t / S_{t-1})
    ↓
ARMA(1,1)-GARCH(1,1)  -- removes autocorrelation and volatility clustering
    ↓
Standardised innovations  ẑ_t = ε_t / σ_t
    ↓
NIG distribution fitting  -- captures residual heavy tails
    ↓
VaR / CVaR forecasts  -- at 95% and 99% confidence levels
    ↓
Backtesting  -- binomial, KS, Anderson-Darling, Christoffersen
```

### Assessment tests

| Test | What it measures | Result |
|------|-----------------|--------|
| Binomial (VaR 99%) | Exceedance count vs expected | ✅ All 5 assets pass |
| Binomial (VaR 95%) | Exceedance count vs expected | ✅ All 5 assets pass |
| Kolmogorov-Smirnov | Max CDF deviation, all quantiles | ✅ NIG best fit on all assets |
| Anderson-Darling | Weighted CDF deviation, tail emphasis | ✅ NIG best fit on 4/5 assets |
| Christoffersen | Exceedance independence (no clustering) | ✅ 7/10 combinations pass |

## Reproducing the analysis

### 1. Clone the repository
```bash
git clone https://github.com/Amos-Anderson/arma-garch-beta-risk-model.git
cd arma-garch-beta-risk-model
```

### 2. Create and activate the environment
```bash
conda create -n beta-risk-env python=3.11 -y
conda activate beta-risk-env
pip install -r requirements.txt
python -m ipykernel install --user --name beta-risk-env --display-name "Beta Risk Model"
```

### 3. Run the notebooks in order
```bash
jupyter lab
```

Open and run notebooks `01` through `06` in sequence. Each notebook 
loads processed data from the previous step via `data/processed/`.

**Runtime estimates:**
- Notebook 01: ~1 minute (data download)
- Notebook 02: ~10–15 minutes (1,000-window rolling GARCH loop × 5 assets)
- Notebook 03: ~2 minutes (NIG fitting + QQ computation)
- Notebook 04: ~3 minutes (VaR/CVaR computation)
- Notebook 05: ~15 minutes (sensitivity analysis re-runs rolling loop × 3 windows)
- Notebook 06: ~2 minutes (figure export)

## Key findings

**NIG vs Gaussian — KS improvement:**

| Asset | Gaussian KS | NIG KS | Improvement |
|-------|-------------|--------|-------------|
| ^GSPC | 0.0602 | 0.0170 | 71.7% |
| AAPL | 0.0613 | 0.0155 | 74.7% |
| GLD | 0.0560 | 0.0115 | 79.4% |
| TIP | 0.0436 | 0.0245 | 43.8% |
| EURUSD=X | 0.0310 | 0.0119 | 61.5% |

**NIG parameters:**

| Asset | $\alpha $ (tail heaviness) | $\beta $ (asymmetry) | $\delta $ (scale) |
|-------|-------------------|---------------|-----------|
| ^GSPC | 1.4017 | −0.3117 | 1.4165 |
| AAPL | 0.7913 | −0.0324 | 0.9602 |
| GLD | 1.2642 | −0.0412 | 1.4349 |
| TIP | 1.5184 | −0.0407 | 1.4933 |
| EURUSD=X | 1.2644 | +0.0209 | 1.3696 |

AAPL has the smallest $\alpha $ (heaviest tails) consistent with single-stock 
return behaviour. ^GSPC has the most negative $\beta $ (strongest left skew) 
consistent with the asymmetric impact of equity market crashes.

## Limitations

- **Symmetric GARCH(1,1)** does not capture the leverage effect. 
  GJR-GARCH or EGARCH would improve ARCH removal for equity assets.
- **Christoffersen failures** at VaR 95% for AAPL, GLD, and EURUSD=X 
  indicate residual exceedance clustering not fully addressed by GARCH(1,1).
- **Univariate modelling** - each asset is modelled independently. 
  A multivariate PCA-copula extension would enable portfolio-level VaR.
- **EURUSD=X convergence** - 13% GARCH convergence failure rate due to 
  prolonged low-volatility FX regimes. A dedicated FX volatility model 
  would improve robustness.
